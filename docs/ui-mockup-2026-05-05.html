Section 01 — 改了什么

  • 顶栏从 TopAppBar 升级为 LargeFlexibleTopAppBarstable 1.4，标题用 Bricolage Grotesque 显示字体，下方 subtitle 显示运行时和工具数 — 信息密度更高、更有"Agent 控制台"的气质。
  • 工具胶囊 在状态切换时做受控形状变化：执行中 = pill（运动感），成功 = 非对称 squircle（沉稳）。官方已给 LoadingIndicator 与部分 Chip 提供形变能力；工具胶囊落地时建议封装 AmberToolCapsule，用 animateContentSize / shape state 收口，而不是把实验 API 散落在业务代码里。
  • Loading 替换 PigLoadingIndicator 旁的 dots，改用 LoadingIndicator / WavyProgressIndicatoralpha 1.5 的波形线，工具卡片下方也用同款表示"正在拉取"。
  • 气泡角形从字面量 RoundedCornerShape(22dp,8dp,…) 收敛到 cookie 形状 token（通过 MaterialShapes 派生），保持当前不规则气泡的辨识度但更柔。
  • ChainOfThought 从纯左侧竖线升级为带容器的 surface-container-low 圆角块，分隔感更强、不再"贴在边上"。

Section 02 — 改了什么

  • 操作行从散落的 chip + icon button，收敛为 ButtonGroupstable 1.4：模型 / 搜索 / MCP 视觉成组，状态压缩成"一个胶囊"。

  • "+" 附件保留现有 ExpandState.Files 折叠行为不动 — 不引入 FAB Menu。只把展开后的每个附件 chip 用统一的 cookie leading icon container（图片 / 拍照 / 文件 / 语音），跟工具胶囊里的 icon-circle 形状语言对齐即可。
  • Composer 形状保留你们原本设计的非对称 32/32/28/14 角形（这个气质不能丢），但把内部按钮的字面 dp 半径替换为统一 shape token。
  • 不动的部分：inputState、messageContent、附件启动器、camera/UCrop、权限、MCP picker、WebView 预览 — 全部不碰，按 handoff 的红线办。

  • Small icon 不变：保留现有的 cutout 矢量（白底 + 透明镂空），它在系统模板着色下是正确的"小猪剪影" — 这是上轮已经修过的，我不打算再动。

Section 04 — 改了什么

  • Hero 状态卡：顶部加了一张 Agent 实时状态卡（cookie 形状 + 装饰背景几何）。当前 settings 进去看不到 Agent 的实时状态，必须返回主对话才知道 — 这是 AmberAgent 作为"应用级 Agent"产品定位下应该被强调的信息。
  • 开关 (Switch)从 Material 2 风格升级为 M3 Expressive Switch — 选中态 thumb 是 pill 形（跟着 ripple 形变），未选中是小圆点。比当前 ListItem 末尾的 chevron 更有"可立即操作"的暗示。
  • CardGroup shape：每组 card 的下两个角用更大的 corner（24/24/32/16），形成"由上至下渐进圆"的视觉节奏 — 这是 M3 Expressive 文档强调的 shape morphing per group 写法。
  • Leading icon container 从 RoundedCornerShape(12dp) 改为 cookie 形状，跟头像 / FAB / 工具胶囊形成统一的"软方"语言。
  • 支持信息更具体：原来 sub text 是分类描述（"核心 / 短期 / 长期"），现在带具体数字（"核心 8 · 短期 24 · 长期 156"）— 信息密度上去了，用户不需要点进去才知道里面有多少东西。

注意事项 · 这只是一个 HTML 仿真

1. Shape morph 真实质感要等真机：CSS 的 transition 只能模拟方向和时长；Compose 侧已在 LoadingIndicator 与部分 Chip overload 中提供形变能力，工具胶囊这种自定义组件要通过 wrapper 控制何时 morph。长对话里同时 morph 30 个工具卡可能掉帧，需要按"状态首次变化时 morph，稳态后回静态"原则收口。

2. WavyProgressIndicator 的真实形态：这里用 SVG dasharray 平移近似，Compose 里的 Wavy API 已在 1.5.0-alpha18 推进；视觉方向一致，但真机上需要确认波幅、触控反馈和低端设备帧率。

3. Alpha API 风险：LoadingIndicator / WavyProgress / shape-morphing chip overloads / expressive theme 相关 API 仍在 1.5.0-alpha18 线。建议每个用到 alpha API 的位置都包一层自家 wrapper Composable（AmberWavyLoading / AmberToolCapsule），收敛未来一次性改名。FAB clover layout 在这里是视觉原型，可用 FloatingToolbar 或自定义 FAB 编组实现，不应当先承诺为现成官方组件。

4. 不要把品牌色当装饰：你看到的琥珀色是 M3 You 派生出来的 hue chroma tone 系统，不是平面 orange。如果偏要换品牌主色（Lavender / Sage 在 Tweaks 里），整套色板会自动重算 — 但不要混用两个色板。
