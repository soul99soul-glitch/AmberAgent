# 伴随智能（Live Companion）重构设计

日期：2026-06-10
状态：已确认（用户批准方案 A + 双模式）

## 背景与现状

伴随智能现状是一个基于无障碍服务的只读屏幕分析器：

- `LiveModeManager`（feature/live，559 行）固定轮询（最小间隔 5~30s），每次抓取
  无障碍 UI tree 文字（正文/树各截 4000 字符），喂给当前聊天模型。
- 五种预设动作（找重点/总结/找下一步/查风险/写回复）+ 自由焦点指令（支持语音）。
- 输出解析成卡片（一句结论 + 要点 + 建议），展示在为 360dp 分屏侧栏设计的
  `LiveCompanionPage`。

用户确认的四个核心痛点（全部命中）：

1. **分析内容泛泛不准** — 模型只看文字树，丢失图片、排版、视觉重点。
2. **只能看不能动，没闭环** — 写好的回复要手动复制粘贴，建议无法执行。
3. **交互太重不伴随** — 要切到专门页面/分屏才能看。
4. **节奏哑** — 固定轮询，屏幕没变也重复分析烧 token，该出声时不出声。

## 用户决策记录

| 决策点 | 结论 |
|---|---|
| 主场景 | 聊天/社交辅助、阅读/信息提炼、办事操作向导（明确不要"全场景常时待命"） |
| 动手边界 | **只填不发**：填输入框/进剪贴板可以，发送/提交永远由用户按 |
| 视觉输入 | 双模式：保守=只读 UI tree；激进=持续截屏喂视觉模型 |
| 路线 | 方案 A 分两期：一期重做引擎，二期上悬浮气泡形态 |

## 设计

### 1. 双模式

| | 保守模式 | 激进模式 |
|---|---|---|
| 输入信号 | 无障碍 UI tree 文字（现有管道） | **截屏为主** + UI tree 辅助 |
| 模型要求 | 任意聊天模型 | 需要视觉模型 |
| 成本/隐私 | 低 | 高 |

- 截屏用 `AccessibilityService.takeScreenshot()`（API 30+，复用现有无障碍服务，
  不引入 MediaProjection 录屏权限）；压缩到长边 ~1280px 后喂模型，控制 token。
- 模式为设置项，页面（二期：气泡）提供快捷切换。
- 激进模式下若伴随模型不支持视觉：提示一次并本次自动降级为保守模式运行，
  不报错、不中断。
- 设备低于 API 30（无 `takeScreenshot`）同样自动降级保守模式并在设置中说明。
- 新增独立"伴随模型"设置项，默认跟随当前聊天模型；允许聊天与伴随使用不同模型
  （例如伴随用更便宜的视觉模型）。

### 2. 事件驱动引擎（双模式共用，替换轮询）

- **触发源**：
  - `TYPE_WINDOW_STATE_CHANGED`（应用/页面切换）
  - `TYPE_WINDOW_CONTENT_CHANGED`（聚合防抖 ~1s 后再判断）
  - 用户主动召唤（点击刷新/气泡/语音/焦点指令）→ force 分析
- **去重**：分析前计算屏幕签名；签名未变则跳过。
  - 保守模式：UI tree 文字 hash。
  - 激进模式：文字 hash + 截图感知哈希（dHash）双重。
- **冷却**：同屏最小分析间隔沿用现有 `minAnalysisIntervalMs`（5~30s clamp）；
  熄屏、回到 Amber 自身界面时挂起，不触发分析。

### 3. 场景画像（按前台应用自动选默认动作）

内置包名分类表（MVP 内置，后期开放用户自定义归类）：

| 类别 | 示例包名 | 默认行为 |
|---|---|---|
| 聊天类 | 微信/QQ/Telegram/钉钉 等 | 自动出**回复草稿**（写回复动作） |
| 阅读类 | 浏览器/知乎/资讯类 app 等 | 自动**提炼重点**（找重点/总结） |
| 其他 | 未归类 app | **静默**，仅用户主动召唤时按"办事向导"（找下一步）分析 |

用户的显式焦点指令始终优先于场景默认动作。

### 4. 只填不发闭环

- 卡片回复草稿带"填入"按钮：在无障碍树中定位焦点 editable 节点（或场景画像
  指定的输入框）→ `ACTION_SET_TEXT` 写入。
- 找不到可填节点：复制到剪贴板 + Toast 提示降级。
- 发送/提交类动作永不代执行。

### 5. 分期交付

**一期（引擎）**——本设计的实施范围：

- 双模式 + 事件驱动引擎 + 场景画像 + 填入闭环；仍用现有 `LiveCompanionPage` 承载。
- `LiveModeManager` 拆为三块（同包内拆文件，不引入新模块）：
  - 引擎：触发/去重/冷却状态机
  - 分析器：prompt 构建 + 卡片解析（现有 `LivePrompt`/`parseCard` 逻辑保留迁移）
  - 动作：填入/剪贴板

**二期（形态）**——另出实施计划：

- 悬浮气泡（`SYSTEM_ALERT_WINDOW` 权限）：静默小点 / 分析中 / 有结果徽标三态；
  点开为卡片（结论/要点/草稿 + 填入按钮）；可拖动贴边。
- 现有页面降级为设置 + 历史页。

### 6. 测试

- 引擎状态机（触发 → 去重 → 冷却 → 分析）纯逻辑单测，事件与时钟可注入。
- 填入动作：找到输入框 / 找不到降级剪贴板两条路径单测。
- 卡片解析沿用现有逻辑与测试，不动。

## 不做的事（YAGNI）

- 不做"可信场景小段自动执行"（用户明确选了只填不发）。
- 不做全场景常时雷达式提醒（用户未选）。
- MVP 不做包名分类的用户自定义 UI（内置表先行）。
- 一期不动悬浮窗。
