  text=Summary
  text=AmberAgent is an AI chat client built on Jetpack Compose. It supports multiple assistants, each with its own system prompt, model parameters, and conversation isolation. Think of each assistant as a separate persona with its own memory and settings.

  text=Architecture Diagram(Prose)
  text=The architecture follows a layered approach:
  text=1.
  text=Surfaces layer — Compose screens that observe StateFlow from ViewModels
  text=2.
  text=ViewModel layer — coordinates UI state and delegates to repositories
  text=3.
  text=Repository layer — single source of truth, backed by Room and remote API
  text=4.
  text=Agent Kernel — pure Kotlin contracts for agent execution and event streaming
  text=Note: The Agent Kernel (core/agent-runtime) has no Android dependencies. It is testable on the JVM without a device.
  text=Example:Starting a Chat
      text=kotlin
      role=Button desc=Save
      role=Button desc=Copy Code
            text=val input = ChatTurnInput(
 assistantId = "assistant_01",
 userMessage = "Explain coroutines in 3 sentences.",
 attachments = emptyList()
)

viewModelScope.launch {
 agentRunner.run(input).collect { event ->
 when (event) {
 is AgentEvent.TextDelta -> appendText(event.delta)
 is AgentEvent.Done -> markComplete()
 is AgentEvent.Error -> showError(event.cause)
 }
 }
}
  text=The agentRunner.run() call returns a Flow<AgentEvent> that the ViewModel collects. Each TextDelta carries a chunk of the assistant's response.
