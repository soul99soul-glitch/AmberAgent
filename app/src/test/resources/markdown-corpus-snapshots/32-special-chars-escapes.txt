  text=Special Characters and Backslash Escapes
  text=CommonMark defines a set of ASCII punctuation characters that can be escaped with a backslash to prevent them from being interpreted as Markdown syntax.
  text=Escapable Characters

  text=Each one above should render as the literal character, not as Markdown syntax.
  text=Practical Escape Examples
  text=A literal asterisk: \*not italic\* — the backslashes prevent emphasis.
  text=A literal underscore: \_not italic either\_.
  text=A literal backtick: \`not inline code\`.
  text=A literal hash at line start:
\# This is not a heading because the hash is escaped.
  text=A literal bracket
  text=$$not a link$$
  text=.
  text=A literal pipe: \| not a table cell separator \|.

  text=Numeric Character References
  text=Decimal: &#65; &#66; &#67; → A B C
  text=Hexadecimal: &#x1F600; → 😀 (emoji via code point)
  text=Literal Backslash
  text=A single backslash followed by a non-punctuation character is not an escape: \a \z \1 — the backslash is kept verbatim.
  text=A literal backslash is produced by \\: here is one → \\
  text=Special Characters in Code Spans
  text=Inside inline code, characters are literal and no escaping is needed: \*not italic\* renders showing the backslashes and asterisks exactly.
  text=URL-Safe Characters
  text=In link destinations (, ), and & need attention:
  text=•
  text=Parentheses in URLs: example — balanced parens are fine.
  text=•
  text=Ampersand in URLs: search — unescaped in Markdown source.
  text=•
  text=HTML entity in URL: entity — entity should be decoded before use as href.
  text=Combining Escapes and Emphasis
  text=A sentence with an escaped asterisk \* followed by actual emphasis and then another escaped asterisk \* — the parser must handle the mix correctly.
