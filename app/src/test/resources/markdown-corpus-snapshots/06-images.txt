  text=Images
  text=Images use the same syntax as links but with an exclamation mark prefix:
  image=[Android robot logo]
  text=The text in brackets is the alt text, which screen readers use to describe the image. The URL in parentheses is the image source.
  text=Images with Titles
  text=Like links, images can have optional titles that appear as tooltips:
  image=[Jetpack Compose logo]
  text=Relative Image Paths
  text=In a real project, images are often local assets:
  image=[Architecture diagram]
  text=The renderer is responsible for resolving relative paths, though in most chat contexts only absolute URLs are meaningful.
  text=Images Inside Links
  text=An image can be used as the clickable content of a link by nesting the image syntax inside link brackets:
  text=Kotlin logo
  text=Clicking the image above would navigate to kotlinlang.org.
  text=Alt Text Matters
  text=The alt attribute is important for accessibility. When the image fails to load, the alt text is displayed in its place. A good alt text describes the image's content, not its appearance — for example, "screenshot of the Settings screen showing dark mode toggle" rather than just "screenshot".
  text=Inline Images in Paragraphs
  text=Images can appear inline within a sentence,
  text=like this satellite view of the Eiffel Tower
  image=[Eiffel Tower thumbnail]
  text=though in practice inline images inside paragraphs can be tricky to style consistently across different devices.
