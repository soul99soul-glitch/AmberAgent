  text=Indented Code Blocks
  text=Before fenced code blocks became the dominant style, Markdown used indentation (4 spaces or 1 tab) to denote code. CommonMark still supports this syntax.
  text=Here is a simple indented code block showing a shell command:
  text=echo "Hello, world!"
 ls -la /usr/local/bin
 export PATH="$PATH:/usr/local/bin"
  text=Note that a blank line and a paragraph precede each indented block, and the code block ends when the indentation stops.
  text=The following example shows an indented Python snippet that might appear in an older tutorial:
  text=def fibonacci(n):
 if n <= 1:
 return n
 a, b = 0, 1
 for _ in range(2, n + 1):
 a, b = b, a + b
 return b

 for i in range(10):
 print(fibonacci(i))
  text=Indented code blocks cannot specify a language, so syntax highlighting is never applied to them. That is one reason fenced blocks with language tags are now preferred.
  text=A paragraph can follow the indented block immediately (after a blank line):
  text=# This is inside the code block
 # Not a Markdown heading
  text=The line above is code, not a heading, because it is indented by four spaces.
  text=You can have multiple separate indented blocks in one document:
  text=first block
 line two
  text=Some prose in between.
  text=second block
 line two of second
  text=The two blocks above are separate code nodes even though they look similar.
